## 📋 Lessons

1. **Sandbox state is ephemeral.** Don't rely on `/app/data-intelligence-architect/`
   persisting across sessions — git push to remote would be safer for any
   work that must survive a wipe. Currently there's no remote configured.
2. **Bash arg parsing gotcha.** `for x in "$@"; do shift; ...` doesn't behave
   like C-style argv parsing. Use positional-index while loops for `--flag VALUE`
   patterns.
3. **tee + redirect = duplicate writes.** When stdout/stderr are both redirected
   to a file, `tee` to that file means double-write. Pick one path.
4. **`for arg in "$@"` snapshot semantics.** Inside the loop, `$@` reflects
   current state (post-shift), but the iteration uses the original snapshot.
   This is a common source of off-by-one in shell scripts.
